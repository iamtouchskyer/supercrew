# i18n Design: Chinese / English Support

**Date:** 2026-02-28
**Branch:** feature/jingxia-kanban
**Scope:** Full-app internationalization — English (default) + Chinese

---

## Context

- `i18next`, `react-i18next`, `i18next-browser-languagedetector` are already installed
- Main app routes (Board, People, Knowledge, Decisions, Task detail) have hardcoded English
- FRE routes (login, auth/callback, welcome) have hardcoded Chinese
- No i18n config or locale files exist yet

---

## Architecture

```
crew/frontend/packages/local-web/src/
  i18n.ts              ← i18next init (side-effect import)
  locales/
    en.json            ← source of truth (English)
    zh.json            ← Chinese translation
```

`Bootstrap.tsx` adds `import '@web/i18n'` at the top to initialize before React renders.

**No I18nextProvider needed** — react-i18next reads from the globally-initialized i18n instance.

---

## Language Detection

Order of precedence:
1. `localStorage['crew-lang']`
2. Browser navigator language (`navigator.language`)
3. Fallback: `en`

Supported languages: `en`, `zh`

## Language Toggle UI

| Location | Behavior |
|----------|----------|
| Sidebar (main app) | GlobeIcon button, click cycles EN ↔ ZH, placed next to theme toggle |
| Login page | `EN \| 中` text toggle in top-right area of the glass card |
| Welcome page | `EN \| 中` text toggle in top-right corner of the Stepper area |
| auth/callback | None (ephemeral page) |

Language change is instant (no page reload), persisted to `localStorage['crew-lang']`.
